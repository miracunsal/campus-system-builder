# Campus System Builder 🏛️💻
**Interactive System Analysis & Design Learning Platform**

[![Live Demo](https://img.shields.io/badge/Live_Demo-Play_Now-06b6d4?style=for-the-badge&logo=github)](https://miracunsal.github.io/campus-system-builder)

An interactive, gamified learning platform developed to teach Software Engineering students the core concepts of **System Analysis and Design**. Originally developed for the COM4061 course, this project allows users to step into the role of a Junior System Analyst and architect a digital university campus from scratch.

## 🚀 Live Demo
You can experience the interactive platform directly from your browser (No installation required):
👉 **[Play Campus System Builder](https://miracunsal.github.io/campus-system-builder)**

## 🧠 Core Educational Concepts
The platform simulates real-world software architecture challenges by replacing traditional theoretical quizzes with hands-on system modeling:
* **Domain Modeling (ER):** Establishing correct entity relationships (One-to-Many, Many-to-Many) for complex database architectures.
* **UML Use Case Diagrams:** Identifying actors, system boundaries, primary goals, and alternative flows.
* **Activity Flows:** Constructing chronological business processes and logical decision nodes.
* **System Scope:** Defining functional and non-functional requirements (Scope-In vs. Scope-Out).

## 🛠️ Technical Architecture & Stack
This project is engineered as a highly optimized, **single-file web application**. It runs a full modern frontend stack entirely in the browser without the need for a Node.js environment or build tools like Webpack/Vite.

* **Frontend Framework:** React 18 (Standalone via CDN)
* **Styling & UI:** Tailwind CSS (with custom animations and glassmorphism UI)
* **JavaScript Compiler:** Babel (In-browser JSX compilation)
* **Animation & Gamification:** Canvas Confetti

## ✨ Key Features
1.  **Real-Time Validation Engine:** Provides immediate, dynamic feedback (Easy, Normal, Analyst modes) when a user selects incorrect actors, entities, or process steps.
2.  **Gamified Progression:** Users earn points, unlock achievements, and upgrade their career titles (from *Junior Analyst* to *Campus Architect*) as they solve architectural problems.
3.  **Dynamic Campus Map:** A visually engaging map that upgrades from "Locked" to "Active" as modules (Student Affairs, Registration, etc.) are successfully integrated.
4.  **Academic Report Generator:** Automatically compiles the user's architectural decisions and performance metrics into a printable, professional PDF summary.
